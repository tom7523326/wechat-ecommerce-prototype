# 设计系统规范 v2.0

## 一、设计原则

### 1.1 核心原则
- **用户为中心**：以学员成长路径为核心设计
- **数据驱动**：全周期数据追踪，可视化呈现
- **生态共建**：从单点培训到生态赋能
- **渐进式披露**：信息分层，按需展示

### 1.2 设计理念
- **简洁高效**：减少认知负担，提升操作效率
- **情感化设计**：成长激励、成就感营造
- **一致性**：统一的视觉语言和交互模式
- **可访问性**：符合WCAG 2.1 AA标准

## 四、布局系统

### 4.1 栅格系统

基于12列栅格：
```css
.container {
    max-width: 1200px;
    margin: 0 auto;
    padding: 0 24px;
}

.grid {
    display: grid;
    gap: var(--spacing-lg);
}

/* 响应式 */
@media (max-width: 768px) {
    .container {
        padding: 0 16px;
    }
}
```

### 4.2 间距系统

使用统一的间距变量，保持视觉一致性。

## 五、交互设计规范

### 5.1 反馈机制

#### 加载状态
- 骨架屏（Skeleton Screen）
- 进度指示器
- 加载动画

#### 操作反馈
- Toast提示（2-3秒）
- 成功动画
- 错误提示

#### 空状态
- 友好的插画
- 引导文案
- 操作建议

### 5.2 动画规范

#### 页面转场
- 淡入淡出：300ms
- 滑动：300ms ease-out

#### 元素动画
- 悬浮：150ms
- 点击：100ms
- 加载：500ms
