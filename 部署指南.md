# 原型部署指南

本指南将帮助您将HTML原型发布为线上链接，方便分享给他人查看。

## 🚀 推荐方案（按简单程度排序）

### 方案一：Netlify Drop（最简单，推荐）

**优点**：无需注册账号，拖拽即用，5分钟完成

**步骤**：
1. 访问 https://app.netlify.com/drop
2. 直接将整个项目文件夹拖拽到页面上
3. 等待上传完成（约1-2分钟）
4. 获得一个免费的 `.netlify.app` 域名链接
5. 分享链接即可

**注意事项**：
- 链接永久有效（只要文件不删除）
- 可以自定义域名
- 支持HTTPS

---

### 方案二：Vercel（同样简单）

**优点**：快速部署，自动HTTPS

**步骤**：
1. 访问 https://vercel.com
2. 使用GitHub账号登录（或注册）
3. 点击 "Add New" → "Project"
4. 选择 "Import Git Repository" 或直接拖拽文件夹
5. 点击 "Deploy"
6. 获得 `.vercel.app` 域名链接

---

### 方案三：GitHub Pages（免费，需要GitHub账号）

**优点**：完全免费，版本控制

**步骤**：
1. 在GitHub创建新仓库（Repository）
2. 将所有文件上传到仓库
3. 进入仓库 Settings → Pages
4. 选择 Source 为 "main" 分支
5. 保存后获得 `https://用户名.github.io/仓库名` 链接

**详细步骤**：
```bash
# 1. 初始化Git仓库（如果还没有）
cd "/Users/tangshoulin/微信电商能力提升"
git init
git add .
git commit -m "Initial commit"

# 2. 在GitHub创建新仓库，然后执行：
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main

# 3. 在GitHub仓库设置中启用Pages
```

---

### 方案四：Surge.sh（命令行工具）

**优点**：快速部署，适合开发者

**步骤**：
```bash
# 1. 安装Surge
npm install -g surge

# 2. 进入项目目录
cd "/Users/tangshoulin/微信电商能力提升"

# 3. 部署
surge

# 4. 按提示输入邮箱和密码（首次使用需要注册）
# 5. 输入域名（或使用随机域名）
# 6. 完成部署
```

---

## 📝 部署前检查清单

- [x] 所有HTML文件都在同一目录
- [x] 文件路径使用相对路径（已确认）
- [x] 入口文件：`原型索引.html` 或 `交互原型-首页.html`
- [x] 所有页面链接正确

## 🔗 部署后的访问方式

部署成功后，访问链接即可：
- 首页：`https://你的域名/交互原型-首页.html`
- 索引页：`https://你的域名/原型索引.html`（推荐作为入口）

## 💡 建议

1. **使用Netlify Drop**：最简单，适合快速分享
2. **设置入口页面**：可以将 `原型索引.html` 重命名为 `index.html`，这样访问域名根目录即可看到索引页
3. **自定义域名**：Netlify和Vercel都支持绑定自定义域名

## ❓ 常见问题

**Q: 部署后链接失效了？**
A: 检查文件是否被删除，或重新部署一次。

**Q: 可以更新内容吗？**
A: 可以，重新上传文件即可更新（Netlify Drop需要重新拖拽）。

**Q: 链接可以分享给多少人？**
A: 无限制，任何人都可以通过链接访问。

**Q: 需要付费吗？**
A: 以上方案的基础版本都是免费的。
